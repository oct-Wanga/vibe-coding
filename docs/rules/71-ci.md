# CI Rules (GitHub Actions)

## 1) 기본 원칙

- CI는 GitHub Actions를 기준으로 운영한다.
- PR 머지 전 반드시 CI 검증 잡이 모두 통과해야 한다.
- 릴리즈 자동화는 `main` 브랜치 push에서만 수행한다.

---

## 2) 필수 검증 잡

- `Web Lint`
- `Web Unit Test`
- `API Unit Test`
- `Web Build`

필수 잡 이름 변경 시, 브랜치 보호 규칙(required status checks)도 함께 갱신한다.

---

## 3) 릴리즈 규칙

- 릴리즈는 `semantic-release`를 사용한다.
- 릴리즈 잡 Node 버전은 `semantic-release` 최소 요구 버전 이상으로 유지한다.
- 커밋 메시지는 Conventional Commits 규칙을 따라야 버전이 올바르게 계산된다.
